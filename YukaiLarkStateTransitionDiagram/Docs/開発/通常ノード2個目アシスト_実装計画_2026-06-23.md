# 通常ノード2個目アシスト 実装計画 2026-06-23

## 方針

- ユカイラークが、［開始マーク］から最初の［通常ノード］への遷移を作ったあと、2個目の［通常ノード］を作るか尋ねる。
- 2個目の［通常ノード］は任意なので、数秒間操作されなければ今回は作らないものとして次のアシストへ進む。
- 2個目の［通常ノード］を作った場合は、［通常ノード］から［通常ノード］への［遷移］作成を続けて提案する。
- ［開始マーク］から最初の［通常ノード］への遷移は従来どおりイベントなし、［通常ノード］同士の遷移はイベント追加アシストの対象にする。
- `YukaiLarkAssistant.cs` が 0 バイトになっていてビルド不能だったため、直近コミットの内容をベースに復旧してから拡張する。

## 実装手順

1. `YukaiLarkAssistant.cs` を復旧し、`CreateSecondStateNode` を追加する。
2. アシスト待機時間を過ぎても 2個目作成が実行されない場合、アシスタント内部でその提案をスキップする。
3. `YukaiLarkAssistantContext` に、開始マークから通常ノードへの遷移有無と、通常ノード同士の遷移有無を追加する。
4. `YukaiLarkAssistOperations` で、2個目通常ノード作成と、状況に応じた遷移作成先の切り替えを実装する。
5. ゴースト表示と完了吹き出しの文言を、2個目通常ノードと通常ノード同士の遷移に対応させる。
6. `dotnet build .\YukaiLarkStateTransitionDiagram.slnx` で確認する。
