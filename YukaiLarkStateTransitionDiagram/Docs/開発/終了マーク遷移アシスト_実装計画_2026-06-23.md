# 終了マーク遷移アシスト 実装計画 2026-06-23

## 方針

- ［終了マーク］があり、［通常ノード］から［終了マーク］への遷移がまだないとき、ユカイラークが遷移作成を提案する。
- 接続元は［開始マーク］からの距離が一番遠い［通常ノード］にする。同距離の場合は ID が小さい方を選ぶ。
- ゴースト表示と実際のアシスト実行で、同じ接続元・接続先を使う。
- ［通常ノード］から［終了マーク］への遷移は、通常どおりイベント追加アシストの対象にする。

## 実装手順

1. `YukaiLarkAssistantContext` に終了マーク向け遷移の有無を追加する。
2. `Game1` のアシストコンテキストと遷移ゴースト接続先を、開始マークから最遠の通常ノードから終了マークへ対応させる。
3. `YukaiLarkAssistOperations` の遷移作成接続先を同じ基準にそろえる。
4. アシスト文言を終了マーク向け遷移に対応させる。
5. `dotnet build .\YukaiLarkStateTransitionDiagram.slnx` で確認する。
