# 開始終了マーク用語統一 実装計画 2026-06-22

## 決定事項

- ユーザー向けの用語は［開始マーク］と［終了マーク］に統一する。
- コード上の英語名は `StartMarker` と `EndMarker` に統一する。
- ［開始マーク］は状態ノードではなく、開始位置を示す固定マークとして扱う方針にする。
- ［終了マーク］も状態ノードではなく、終了位置を示すマークとして扱う方針にする。
- `N` キーは通常の［ノード作成］、`E` キーは［終了マーク］作成に分ける方針にする。

## 今回の第一段階

1. 説明書と画面文言の「開始ノード」「終了ノード」を「開始マーク」「終了マーク」へ寄せる。
2. コード識別子の `StartNode` / `EndNode` を `StartMarker` / `EndMarker` へ寄せる。
3. `NodeKind.Start` / `NodeKind.End` を `NodeKind.StartMarker` / `NodeKind.EndMarker` へリネームする。
4. 保存 JSON の `Kind` は現状数値なので、enum の順序は `Normal = 0`, `StartMarker = 1`, `EndMarker = 2` を維持する。
5. ビルドで破損がないことを確認する。

## 今回はまだ変えないこと

- ［開始マーク］を最初から 1 個固定で作成する挙動。
- ［開始マーク］の追加・削除禁止。
- `T` キーによる通常ノードから開始/終了への切り替え廃止。
- `E` キーによる［終了マーク］作成。
- ［開始マーク］から出る遷移を 1 本に制限する挙動。
- ［終了マーク］から出る遷移を禁止する挙動。
