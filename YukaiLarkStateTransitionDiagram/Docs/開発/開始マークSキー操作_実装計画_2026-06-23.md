# 開始マーク S キー操作 実装計画 2026-06-23

## 方針

- ダイアグラムに［開始マーク］が無いとき、`S` キーでマウスカーソル位置へ［開始マーク］を追加する。
- ダイアグラムに［開始マーク］が既にあるとき、`S` キーで［開始マーク］が画面中央に来るよう表示位置を移動する。
- `Ctrl+S` は従来どおり保存に使い、素の `S` だけを開始マーク操作に割り当てる。
- 開始マーク追加はダイアグラム変更なので Undo 対象にする。
- 開始マークへの表示移動はダイアグラム内容を変えないので Undo 対象にしない。

## 実装手順

1. `Game1.HandleKeyboard` に `S` キーの分岐を追加する。
2. `HandleStartMarkerShortcut` を追加し、開始マークの有無で作成と表示移動を分ける。
3. `AddStartMarker` を追加し、既存開始マークがある場合は重複作成しない。
4. `CenterViewOnWorldPosition` を追加し、ワールド座標がビューポート中央へ来るよう `_cameraOffset` を更新する。
5. 下部ショートカット表示に、開始マークの有無に応じて `S: 開始マーク追加` または `S: 開始へ移動` を表示する。

## 確認項目

- 開始マーク無しの状態で `S` を押すと、マウス位置に［開始マーク］が 1 個追加される。
- 開始マーク有りの状態で `S` を押すと、開始マークが画面中央に来る。
- `Ctrl+S` は保存のまま動く。
- ラベル編集中、PNG 出力範囲選択中、最近ファイルメニュー表示中は `S` の開始マーク操作が割り込まない。
- `dotnet build .\YukaiLarkStateTransitionDiagram.slnx` が成功する。
